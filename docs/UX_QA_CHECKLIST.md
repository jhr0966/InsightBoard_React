| Python compile/pattern/secret scan | 모든 tracked Python compile 성공, 금지 패턴 없음, `.env.example` 키 유출 없음 | ✅ `make check` 통과 |

## 3. 수동 QA 시나리오

### 3.1 오늘의 보드

- [ ] 페르소나 미설정 시 사이드바 설정 CTA가 보인다.
- [ ] 페르소나 설정 후 부서/직무 기반 카드 문구가 자연스럽다.
- [ ] 뉴스/로드맵/본문 확보 수치가 실제 저장 데이터와 일치한다.
- [ ] `추천 다음 행동`이 데이터 준비 상태에 따라 수집/로드맵/Enrich/SOLA 행동을 우선순위로 보여준다.
- [ ] `자동화 기회 Top 5` 카드가 내 부서 항목을 강조한다.

### 3.2 데이터 관리

- [ ] 데이터 준비 상태 대시보드가 뉴스 DB, 본문 Enrich, 로드맵 DB, LLM 상태를 한눈에 보여준다.
- [ ] 뉴스가 없을 때 수집 필요 안내가 명확하다.
- [ ] 로드맵이 없을 때 업로드 필요 안내가 명확하다.
- [ ] 뉴스 수집 탭 StepGuide가 `키워드·소스 선택 → 수집·저장 → 본문 Enrich → 분석으로 이동` 순서를 보여준다.
- [ ] 로드맵 업로드 탭 StepGuide가 `엑셀 선택 → 시트 확인 → 검증·저장 → 매칭 준비` 순서를 보여준다.

### 3.3 인사이트 분석

- [ ] 분석 실행 흐름 StepGuide가 `트렌드 확인 → 로드맵 연결 → 기회 선별 → SOLA 제안` 순서를 보여준다.
- [ ] 자동화 기회 카드의 `SOLA 제안` 버튼이 SOLA 작업실 제안서 생성 모드로 이동한다.
- [ ] 이동 후 부서/Lv3 필터가 선택한 기회와 일치한다.
- [ ] 사이드 SOLA 컨텍스트에 상위 자동화 기회 후보가 포함된다.

### 3.4 SOLA 작업실

- [ ] 작업 유형 카드가 뉴스 요약, 자동화 과제 제안서, 컨텍스트 채팅, 산출물 보관함 역할을 설명한다.
- [ ] 뉴스/로드맵/LLM 준비 부족 상태가 readiness card에 표시된다.
- [ ] 뉴스 요약 결과를 Markdown으로 다운로드할 수 있다.
- [ ] 뉴스 요약 결과를 산출물 보관함에 저장할 수 있다.
- [ ] 자동화 과제 제안서 생성 후 북마크 저장이 가능하다.

### 3.5 산출물 보관함 / 제안서 작업장

- [ ] 보관함 KPI가 전체 산출물, 제안서, 채택 과제, 검토 중 상태를 표시한다.
- [ ] 제안서 상태(`검토 중`, `채택`, `거절`)와 결정 메모를 저장할 수 있다.
- [ ] 제안서 카드의 `작업장` 버튼이 SOLA 작업실 제안서 작업장 수정 모드로 이동한다.
- [ ] 작업장에서 수정한 본문을 `원본 업데이트`로 기존 북마크에 반영할 수 있다.
- [ ] 작업장 현재 버전을 별도 북마크로 저장할 수 있다.
- [ ] Markdown 다운로드가 정상 동작한다.

## 4. 남은 리스크/후속 권장

1. **브라우저 스크린샷 QA**: 현재 자동화 smoke는 서버 기동/health 확인 수준이다. 실제 브라우저 렌더링 스크린샷은 Playwright 등 브라우저 도구가 있는 환경에서 추가 수행한다.
2. **실데이터 E2E**: 네이버/구글/테크 사이트 수집은 외부 네트워크와 API 상태에 영향을 받으므로 실제 운영 키워드로 별도 점검한다.
3. **LLM 실호출 QA**: `.env`의 LLM 설정이 있는 운영 환경에서 요약/제안서/채팅 결과 품질을 검수한다.
4. **용어 최종 통일**: `북마크`, `산출물`, `제안서 작업장`, `보관함` 용어가 사용자에게 혼동 없는지 최종 리뷰한다.
