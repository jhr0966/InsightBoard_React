## 설계 원칙 (개편으로 확립)

- **결정적 데이터 축**: 정렬·정체성·매칭·랭킹은 버전 상수(IDENTITY/MERGE/MATCHING/TAXONOMY/RANKING_VERSION)를 가진 결정적 계산 — 기사×사용자 LLM 대량 호출 금지. LLM 은 요약·제안서·사례 추출 등 산출물에만.
- **뉴스의 자산화 수직 흐름**: 기사 → (links 매칭) → 사례 추출 → 사람 검토(승인) → **승인 사례만** 제안서 주근거 → 과제 엔터티(이력 보존).
- **이관은 원본 보존·멱등·정직**: bookmark→proposal 이관본은 `legacy` 표시, 근거 없으면 `evidence_unavailable` — 근거 있는 척 하지 않는다.
- **X-User-Id 는 인증이 아니다**: 신뢰 프록시 주입 전제(I-17). 파일 저장소는 단일 서버 파일럿 한정 — 실서비스는 repository seam(I-8) 경유 DB 전환.
